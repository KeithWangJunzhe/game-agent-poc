# Phase 1 Plan

这个文档记录 `game-agent-poc` 的阶段一目标。阶段一的原则是：

- 让游戏本身足够有策略味
- 让最小 harness 能稳定跑通
- 让后续测试有明确指标
- 先不追求复杂可视化，也不追求完整 RL

## 阶段一目标

1. 把胜利条件从“打掉敌方 base”扩展为“先拿到 10 金”
2. 让资源采集、骚扰、偷家、压制农民这些选择都真正有意义
3. 随机分配先后手，保留先手优势作为可观察的策略信号
4. 给 `e2b` 一个足够小但可复现的 harness
5. 保留回放、统计、失败样本记录

## 玩法方向

阶段一希望形成一个最小但有决策张力的局面：

- 采集资源可以直接影响胜利进度
- 攻击 worker 可以打断对方经济
- 偷家可以作为快速结束对局的高风险路线
- 正面推进可以作为中后期压制手段

这样 agent 不会只有一种明显最优解，而是需要在不同目标之间权衡。

## 胜利条件

建议阶段一使用双条件或主条件 + 终止条件：

- 主胜利条件：任意一方先累计到 `10 gold`
- 可选补充条件：敌方 `base` 被摧毁也可直接结束

如果后续想观察更纯粹的经济博弈，可以临时关闭 `base` 秒杀胜利，只保留 `10 gold` 胜利。

## 先手优势

当前 POC 的先手优势不作为阶段一的平衡目标，但需要通过随机分配先后手来保留它作为可观察变量：

- 每局随机决定谁先手、谁后手
- 保留先手是否能形成稳定优势的观察空间
- 如果模型真有策略能力，它应该能逐步识别并利用先手优势
- 后续统计里可以单独看先手 / 后手胜率

我们不在这里先做系统性平衡，只记录它，并把它当成后续策略分析的数据点。

## 最小 harness

阶段一的 harness 不追求复杂，最低只需要：

1. 输入结构化 state
2. 输出一个合法 command
3. 做命令校验
4. 记录每步 action / result / state
5. 生成 match report 和 after-match stats

如果模型输出不合规，harness 可以：

- 重试一次
- 或退回到 baseline / end_turn

## 需要的观测信息

给 agent 的观测至少包括：

- 当前回合
- 当前行动方
- 双方资源
- 单位位置、血量、所属方
- 资源点剩余量
- 当前单位的合法移动方向
- 最近事件摘要
- 当前可用动作集合

## 阶段一要验证什么

阶段一不是为了证明模型很强，而是为了验证：

- agent 是否会围绕资源目标做选择
- agent 是否会把攻击和经济联系起来
- agent 是否能在简单规则下维持局内策略一致性
- harness 是否能稳定记录、回放、统计
- 先手优势是否已经足够被削弱到可测

## 暂不做的事

- 不做 GUI
- 不做复杂地图编辑器
- 不做大型技能树
- 不做真正的模型参数训练
- 不做多智能体协同
- 不做复杂视觉输入

## 推荐推进顺序

1. 先改胜利条件和初始布局
2. 再补最小策略目标
3. 再补最小 harness 接口
4. 再跑对照测试
5. 再决定是否继续加状态机 / skills
