# Post POC Plan

这份文档记录 `game-agent-poc` 完成后，后续怎么使用这套原型。

## 当前结论

- 这一版 POC 的目标已经完成
- 最小 CLI 对局已经跑通
- 最小 harness 已验证可用
- debug trace 和 after-match analysis 已经能支撑基本复盘
- 这个仓库后续更适合作为“方法验证原型”而不是继续无限加功能

## 后续怎么用这套 POC

1. 如果未来更大的项目推进不顺，可以回到这里做最小化测试
2. 可以继续用这套 harness 验证新 prompt、状态压缩、行动空间设计
3. 可以用这套 analysis 看新的 agent 方案是否真的比 baseline 好

## 下一阶段方向

- 选择一个更真实、更复杂、你真正愿意长期投入的游戏
- 优先考虑通过 mod、插件或脚本方式接入 agent 能力
- 先做最小集成，再逐步增加环境信息和 action space
- 等需要时，再回到这个 POC 复用思路、指标和调试工具

## 复用原则

- 保留最小可跑闭环
- 优先保留可观测性
- 不把原型直接膨胀成大工程
- 让它继续作为“回退点”和“验证台”
